Key Findings for dotnet

  • Maintainer squeeze: Across all dotnet repos, open issues and PRs per maintainer are rising. runtime lost ~20% of active maintainers since late 2023, yet workload hasn't shrunk proportionally.
  • Community contributors concentrating: runtime community PR authors dropped ~22%, but PR volume held steady — fewer people contributing more each (7.4 → 8.8 PRs/person). The remaining contributors are more committed, but the funnel is narrowing.
  • First-timer retention cratering: runtime's 12-month return rate dropped from 40% (2020) to 22% (2024). 60% of community authors make exactly one PR and never return. maui's 8× merge latency gap likely drives this — first-timers don't wait.
  • maui Syncfusion effect: 22 Syncfusion engineers began contributing mid-2024, now accounting for 74% of community PRs. Community health metrics for maui are heavily dependent on this single partnership.
  • Issue turnaround holding: Despite fewer maintainers, runtime's community issue turnaround (~50% closed within 30 days) has held steady since 2020. But the open backlog % is creeping up (14% → 30% still open).
  • Copilot adoption accelerating: runtime is the most aggressive adopter of Copilot SWE Agent PRs among dotnet repos, with rapid month-over-month growth across all four repos.
  • Community issue openers declining: runtime and maui both see fewer distinct community issue openers, likely reflecting product maturation — but could also signal disengagement if the open backlog keeps growing.
  • How dotnet compares: vscode has 2× the maintainer pool and closes 60% of issues within 30 days. rust merges ~160 PRs/week via automated bors bot with 1,000+ contributors. dotnet repos are healthy but less automated and more maintainer-constrained than peers.
