# Database Schema ‚Äî pr-dashboard.db

Generated from live database. SQLite with WAL mode.

## Tables

### `items` ‚Äî 805,776 rows

Core table: every issue and PR across all tracked repos.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| `repo` | TEXT | NOT NULL | ‚Äî | e.g. `dotnet/runtime` |
| `number` | INTEGER | NOT NULL | ‚Äî | Issue/PR number within repo |
| `created_at` | TEXT | yes | ‚Äî | ISO 8601 timestamp |
| `closed_at` | TEXT | yes | ‚Äî | NULL if still open |
| `state` | TEXT | yes | ‚Äî | `OPEN` or `CLOSED` |
| `is_pull_request` | INTEGER | NOT NULL | 0 | 1 = PR, 0 = issue |
| `merged_at` | TEXT | yes | ‚Äî | NULL for issues; set for merged PRs |
| `labels` | TEXT | yes | ‚Äî | JSON array of label names |
| `author` | TEXT | yes | ‚Äî | GitHub username who opened the item |
| `merged_by` | TEXT | yes | ‚Äî | GitHub username who merged (PRs only) |
| `copilot_requester` | TEXT | yes | ‚Äî | If PR was authored by Copilot, the human who requested it |

**Primary key:** `(repo, number)`

#### Indexes
- `idx_items_repo_type` on `(repo, is_pull_request)`
- `idx_items_created` on `(repo, created_at)`

## Backup / Restore

- **Backup:** `data/items.csv.gz` ‚Äî gzipped CSV of the full `items` table
- **Restore:** `python load_csv.py` ‚Äî recreates DB from CSV (DB must not exist)
- **DB file** (`pr-dashboard.db`) is gitignored; CSV is committed
- `fetch_progress` is regenerated automatically from item counts during restore

## Scripts that populate this DB

| Script | What it fetches | API | Columns populated |
|--------|----------------|-----|-------------------|
| `fetch.py` | All issues + PRs (core fields) | REST | repo, number, created_at, closed_at, state, is_pull_request, merged_at, labels, author |
| `fetch_mergers.py` | merged_by for merged PRs | GraphQL | merged_by |
| `fetch_copilot_requesters.py` | Copilot requester for Copilot PRs | GraphQL | copilot_requester |
| `fetch_issue_authors.py` | Author for issues (backfill) | REST | author (for issues only) |
